# Story 1.1 — Infraestrutura: Schema Supabase + Config Central

## Status
Ready

## Contexto
Com 9 distribuidores validados e engines a construir, precisamos de um storage persistente para as coletas brutas e um lugar seguro para guardar tokens dinâmicos (especialmente o Fotus). O Gist atual resolve o token do Fotus, mas o Supabase centraliza tudo e permite consultas futuras.

## Objetivo
Criar o schema Supabase com as tabelas `coletas_raw` e `config`, configurar as credenciais como GitHub Secrets, e adaptar o hub_config para ler do Supabase em vez de arquivo local.

## Acceptance Criteria

- [ ] Criar projeto no Supabase (free tier)
- [ ] Criar tabela `coletas_raw` com colunas: id, distribuidor, coletado_em, status, total_itens, erro_msg, payload (jsonb), engine_versao
- [ ] Criar tabela `config` com colunas: chave (PK), valor, expira_em, atualizado_em
- [ ] Inserir token Fotus atual na tabela `config` com chave `fotus_token`
- [ ] Adicionar SUPABASE_URL e SUPABASE_KEY como GitHub Secrets
- [ ] Criar `etapa-2/scripts/supabase_client.py` — módulo utilitário para leitura/escrita
- [ ] Testar: inserir registro de teste em `coletas_raw` e ler de volta
