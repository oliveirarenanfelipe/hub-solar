# Story 3.1 — Módulo de Cotação: Protótipo Streamlit

## Status
Ready

## Contexto
Na hora de comprar (não apenas exibir no hub), o integrador precisa de um preço exato: frete para o CEP real, string box incluída ou não, estoque disponível. O módulo de cotação recebe os parâmetros do kit desejado e retorna uma tabela comparativa de todos os fornecedores com preço total real.

Diferença do hub: o hub é vitrine de preços estimados. Este módulo é para decisão de compra — máxima precisão.

## Objetivo
Criar protótipo Streamlit funcional que recebe potência mínima, inversor, estrutura e CEP, consulta todos os fornecedores disponíveis em tempo real e retorna tabela comparativa com preço total (produto + frete + string box).

## Acceptance Criteria

- [ ] Interface Streamlit com campos: potência mínima (kWp), inversor (com opção "qualquer equivalente"), tipo de estrutura, CEP de entrega
- [ ] Consulta em paralelo todos os fornecedores com API funcionando (mínimo: Nexen, Megacomm, Suno, Helte, BYD)
- [ ] Resultado: tabela com distribuidor, preço kit, frete estimado, string box (inclusa/não), total, prazo (se disponível)
- [ ] Kits abaixo da potência mínima excluídos automaticamente
- [ ] Filtro de oversizing aplicado (0.75 – 1.35)
- [ ] Melhor opção destacada (menor total)
- [ ] Aviso explícito quando frete for FOB (BYD, Aldo, Solfácil) — precisa calcular à parte

## Regras de Matching

| Componente | Regra |
|---|---|
| Potência | Kit resultante >= potência mínima solicitada |
| Inversor | Exato primeiro; se não houver, sugerir equivalente técnico (mesma potência nominal, mesma fase, mesma tecnologia) com flag "equivalente" |
| Estrutura | Exatamente o tipo solicitado — excluir kit se não disponível |
| Painéis | Qualquer painel que entregue a potência |
| String box / cabos | Seguir regra de cada fornecedor automaticamente |
