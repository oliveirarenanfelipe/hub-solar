# Story 1.2 — Engines de Coleta: GitHub Actions para 9 Distribuidores

## Status
Ready

## Contexto
Temos 9 distribuidores com fórmulas validadas (Nexen, Megacomm, Suno, Forthe, Helte, BYD, Aldo, Serrana, Solfácil). Precisamos de engines que rodem automaticamente 2x por semana, coletando kits e salvando no Supabase. Fotus fica separado por depender do token keeper.

## Objetivo
Criar 1 GitHub Actions workflow por distribuidor com matrix jobs (inversor × faixa de potência F1-F4), salvando coletas brutas no Supabase e alertando por Telegram em caso de falha.

## Acceptance Criteria

- [ ] 9 workflows criados (1 por distribuidor validado, exceto Fotus)
- [ ] Cada workflow roda Terça + Sexta, escalonado de 30 em 30 min (horários UTC definidos)
- [ ] Cada job salva em `coletas_raw` com status sucesso/erro/parcial
- [ ] Falha após 3 tentativas → alerta Telegram 🔴 com nome do distribuidor + mensagem de erro
- [ ] Zero kits coletados mesmo com status 200 → alerta 🟡
- [ ] Filtro de oversizing aplicado antes de salvar (ratio 0.75 – 1.35)
- [ ] `engine_versao` incluído em cada coleta para rastreabilidade
